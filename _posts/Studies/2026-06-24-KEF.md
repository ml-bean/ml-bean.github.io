### 고전적 정량 지표 (통계·자동 지표)

| 계열 | 지표 | 측정 방식 | 정답 필요 | 한계 |
| --- | --- | --- | --- | --- |
| 언어모델링 | Perplexity | 시퀀스 예측 확률 (낮을수록 좋음) | ✕ | 확률값 필요, 환각·품질과 무관 |
| 어휘 겹침 | BLEU | n-gram 정밀도 + brevity penalty | ○ | 표면 매칭만, 의미 무시 |
|  | ROUGE | n-gram 재현율 (요약용) | ○ | 〃 |
|  | METEOR | 동의어·어간까지 고려한 P/R 균형 | ○ | 〃 (다소 완화) |
| 분류형 | Accuracy, Exact Match, F1 | 정답과 일치 여부/부분 점수 | ○ | 객관식·추출형에만 적합 |
| 임베딩 기반 | BERTScore, BLEURT, COMET | 문맥 임베딩 유사도 / 학습형 품질 예측 | ○ | 느림, 판정 모델 의존 |
| 검색 랭킹 | MRR, Recall@k, NDCG | 정답 문서의 순위 | ○ | RAG 검색 단계 전용 |

→ **용도**: 정답이 명확한 태스크(번역·요약·추출·검색), 파인튜닝/양자화 전후 성능 저하 측정의 공통 잣대.

### SOTA 프론티어 모델 비교 지표 (범용 능력)

| 평가축 | 벤치마크 | 무엇을 보나 |
| --- | --- | --- |
| 과학·수학 추론 | GPQA Diamond, AIME, FrontierMath, ARC-AGI-2, HLE | 박사급 추론, 미학습 논리, 경시 수학 |
| 실전 코딩 | SWE-bench Verified/Pro | 실제 GitHub 이슈 수정 + 유닛테스트 통과 |
| 도구·에이전트 | BFCL, Terminal-Bench, OSWorld | 함수 호출, 터미널, 컴퓨터 사용 |
| 환각·사실성 | FACTS Grounding, AA-Omniscience | 근거 기반 사실 정확도, 환각률 |
| 인간 선호 | Chatbot Arena (Elo) | 블라인드 비교 선호 |
| 효율성 | 비용/토큰, 속도, 컨텍스트 길이 | 운영 경제성 |

→ **용도**: 범용 지능의 세대 비교. MMLU·HumanEval 등은 포화되어 퇴출. 단일 승자 대신 "작업별 승자" 프레임.

### 기업 특화 sLLM 평가 (도메인·배포 맥락)

말씀하신 대로 2번 지표는 여기서 거의 무력합니다 — "리뷰 답글을 잘 쓰는가"를 GPQA로 잴 수 없으니까요. 대신:

**공통 5축 (자체 사용형·공급형 공통)**

| 축 | 정량 | 정성 |
| --- | --- | --- |
| 업무 정확도 | 자체 골드셋 기반 Task Success Rate (rubric 이진 채점 + pass@k 반복 측정) | 전문가 rubric 채점, LLM-as-Judge |
| 환각 | Faithfulness (지지 진술 비율), NLI claim 검증, 환각 차단율 % | 인간 평가 + Judge 교차검증 |
| 도메인 전문성 | 도메인 벤치마크(KMMLU 분야별, KRX-Bench류), 자격시험 통과 | 전문용어·업무 문맥 전문가 리뷰 |
| 보안성 | 프롬프트 인젝션 방어율, PII 유출 0건, 레드팀 통과율 | 보안 감사·인증(TTA CAT 등) |
| 효율성 | Latency(p99 TTFT/TPOT), Throughput(TPS), 양자화 후 성능 저하율, 전력·메모리 | 사용자 체감 피드백 |
